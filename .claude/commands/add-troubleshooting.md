`$ARGUMENTS` 를 파싱하여 첫 번째 인자를 파일명, 나머지를 증상 설명으로 처리합니다.

## 작업 순서

1. 해당 파일의 **3. 트러블슈팅** 섹션을 읽습니다
2. 아래 형식으로 새 트러블슈팅 항목을 작성합니다:

```markdown
#### 증상
- {사용자가 설명한 증상을 구체적으로 기술}
- 관련 에러 메시지 예시 포함

#### 원인
- 근본 원인을 기술 (추측이 아닌 실제 동작 원리 기반)

#### 해결 방법
```bash
# 진단 명령어
cilium status
kubectl describe pod <POD_NAME> -n <NAMESPACE>
hubble observe --namespace <NAMESPACE>

# 해결 명령어
{해결 kubectl / cilium 명령어}
```

> **예방책**: 재발 방지를 위한 설정이나 모니터링 방법
```

3. 기존 트러블슈팅 섹션의 **3.1 주요 이슈** 또는 **3.2 Q&A** 중 적절한 위치에 추가합니다
4. 추가할 내용을 먼저 보여주고 사용자 확인 후 파일에 반영합니다
