아래 기준으로 `$ARGUMENTS` 문서를 검토하고 개선점을 제안해 주세요.

## 검토 항목

### 1. 구조 완성도
- [ ] 5개 필수 섹션(개요/설명/트러블슈팅/모니터링/TIP) 모두 존재하는지
- [ ] 각 섹션이 충분한 내용을 담고 있는지 (너무 짧으면 보완 필요)

### 2. 코드 품질
- [ ] 모든 코드 블록에 언어 태그가 있는지 (`yaml`, `bash`)
- [ ] 플레이스홀더가 `<VALUE>` 형식으로 명확히 표시됐는지
- [ ] YAML 예시에 `namespace` 가 명시됐는지
- [ ] `kubectl apply` 예시에 `-n <namespace>` 가 포함됐는지

### 3. 보안 검토 (`rules/security-checklist.md` 기준)
- [ ] CiliumNetworkPolicy 예시가 최소 권한 원칙을 따르는지
- [ ] Default-deny 정책이 권장되는지
- [ ] WireGuard 암호화 권장 여부가 명시됐는지

### 4. 모니터링 섹션
- [ ] 관련 `cilium` CLI 진단 명령어가 포함됐는지
- [ ] Hubble/Prometheus/Grafana 확인 방법이 있는지

### 5. 한국어/영어 병기
- [ ] 기술 용어에 영어 원문이 병기됐는지

## 출력 형식

검토 결과를 아래 형식으로 출력하세요:

```
## 검토 결과: {파일명}

### 잘된 점
- ...

### 개선 필요 사항
1. [섹션명] 문제 설명 → 개선 방안

### 보안 이슈 (있을 경우)
- 위험도: High/Medium/Low
- 내용: ...
```

검토 후 사용자 확인을 받아 수정 작업을 진행할지 물어보세요.
